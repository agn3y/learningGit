Git and Github

git add : to add files to stagging

git commit -m "message here": to commit the change with a detailed text message

git log : to see log of current branch

git log --all : to see log of whole repository

git log -all --oneline : all log in single line

git branch : to see list of branch and currently working branch

git branch "nameHere": to create new branch

git branch --delete branch-name : to delete branch

git checkout : to switch branch (multiple use but let's just keep this one for right now)

git merge "branchName" : go to the branch where you want to merge the code with and use comand

git remote show origin : to see with which repo you currently working with

git remote add origin https://github.com/agn3y/learningGit.git : it will connect/set the repository

git branch -M "nameHere" : it will rename the name of branch

git push -u origin main :

Every first-time we need to use this command after that we will be able to use git push, here "main" is the name of the branch. -u option tells Git to set an upstream branch. It establishes a connection between your local branch and the remote branch, so Git knows where to push or pull changes from in the future. This is the default name for your remote repository. When you clone a repository, Git automatically names the remote repository origin. You can think of it as a shorthand for the URL of the remote repository (e.g., https://github.com/user/repository.git).

We can push multiple branches using above given command

git clone : to clone repository

git fetch : download the latest changes from the remote repository, but it will not merge or modify your working directory. git fetch gives you the opportunity to inspect the fetched changes before merging them, which can be helpful for reviewing code or checking for potential merge conflicts. after we use this command we need to merge repo by using git merge command

git pull : fetch changes from a remote repository and automatically merge them into your current branch, inshort git pull will auto mate task which is done by fetch and merge

whenever we use either of them commands we need to define name of the remote repository. something like "origin main"

there is a rule in Git which is not to rewrite or modify history

when we have made a commit by accident or may be we sometime just want to revert the change which are not pushed on repository yet we can use git reset --hard "comitID"
