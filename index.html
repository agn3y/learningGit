How to Resolve Conflict in Git?

So, when does a Conflict occurs ? when we are combining code of one branch into another branch And A Git conflict happens when two people change the same part of a file, and Git doesn’t know which change to keep. It usually happens when both edits affect the same line or are incompatible. Git marks the conflict, and you must choose which version to keep or combine them. Once resolved, you tell Git to continue.

What is Stashing in Git?

Stashing in Git is a feature that allows you to temporarily save changes that you’ve made to your working directory, so you can switch to another branch or work on something else without committing your changes. It’s useful when you’re in the middle of working on something, but need to save your progress and move to another task.

git stash -m "message" : will stash the changes and you can also create a message to remember

git stash list : list of your stashed changes in Git

git stash show -p stash@{ID} : If you want more detailed information about the contents of a particular stash

git stash apply : To apply the most recent stash without removing it from the stash list

git stash apply stash@{1} or 1 : If you want to apply a specific stash

git stash pop : If you want to apply the most recent stash and remove it from the stash list

git stash pop stash@{1} : To pop a specific stash

git stash clear : If you want to delete all stashes

git stash drop stash@{0} : you can delete a specific stash in Git

How to Cherry Pick a Commit in Git?

To cherry-pick a commit in Git means to apply the changes from a specific commit from one branch to another. This is useful if you want to take a single commit from one branch and apply it to your current branch without merging the entire branch.

git cherry-pick : the command to cherry-pick

git cherry-pick "start-commit".."end-commit" : If you want to cherry-pick a range of commits

What is .gitignore file?

A .gitignore file is a special file in a Git repository that tells Git which files or directories to ignore when you are staging, committing, or pushing changes to a repository. Essentially, it prevents certain files from being tracked by Git.
